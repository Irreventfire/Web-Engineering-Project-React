Inspektionsmanagement System

Projektübersicht

Das Inspektionsmanagement System ist eine moderne Webanwendung zur Verwaltung und Durchführung von Inspektionen in industriellen Anlagen. Das System ermöglicht die strukturierte Erfassung, Dokumentation und Auswertung von Prüfungen nach vordefinierten Checklisten.

Hauptfunktionen

1. Benutzerverwaltung

  • Rollensystem: Admin, User und Viewer mit unterschiedlichen Berechtigungen
  • Benutzerkontenverwaltung: Erstellen, Bearbeiten und Deaktivieren von Benutzerkonten
  • Sichere Authentifizierung: BCrypt-verschlüsselte Passwörter
  • Mehrsprachige Oberfläche: Deutsch und Englisch

2. Checklisten-Management

  • Erstellen und Verwalten von Inspektionschecklisten
  • Hinzufügen von Prüfpunkten mit Beschreibungen
  • Upload von Referenzfotos für gewünschte Zustände
  • Flexible Strukturierung nach Anlagenbereichen

3. Inspektionsdurchführung

  • Planung und Zuweisung von Inspektionen
  • Schrittweise Abarbeitung von Checklisten
  • Bewertung: Erfüllt / Nicht erfüllt / Nicht zutreffend
  • Kommentarfunktion für Bemerkungen
  • Foto-Upload des aktuellen Zustands
  • Vergleich mit Referenzfotos

4. Reporting und Auswertung

  • Übersichtliches Dashboard mit Statistiken
  • Detaillierte Inspektionsberichte
  • Status-Tracking (Geplant, In Bearbeitung, Abgeschlossen)
  • Export-Funktionalität

Architektur

Frontend-Architektur

  • Component-Based: Modulare React-Komponenten für Wiederverwendbarkeit
  • Context API: Zentrales State Management für Authentifizierung und Sprache
  • Protected Routes: Rollenbasierte Zugriffskontrolle
  • Responsive Design: Mobile-first Ansatz mit modernem UI

Backend-Architektur

  • RESTful API: Klare API-Struktur mit standardisierten Endpoints
  • MVC Pattern: Controller, Service, Repository Schichten
  • JPA Entities: Objekt-relationales Mapping
  • File Upload: Multipart-Datei-Handling für Bildupload

Datenbank-Schema

Hauptentitäten

  • User: Benutzerdaten, Rollen, Authentifizierung
  • Checklist: Inspektionschecklisten mit Namen und Beschreibung
  • ChecklistItem: Einzelne Prüfpunkte mit Referenzfotos
  • Inspection: Inspektionsaufträge mit Anlage, Datum, Verantwortlichem
  • Result: Prüfergebnisse mit Status, Kommentaren und Fotos

Beziehungen

  • One-to-Many: Checklist → ChecklistItems
  • Many-to-One: Inspection → Checklist
  • One-to-Many: Inspection → Results
  • Many-to-One: Result → ChecklistItem

Sicherheitskonzept

  • Passwort-Hashing: BCrypt mit Salt für sichere Passwortspeicherung
  • Rollenverwaltung: Granulare Berechtigungssteuerung
  • CORS-Konfiguration: Kontrollierter Cross-Origin-Zugriff
  • Input-Validierung: Serverseitige Validierung aller Eingaben
  • Persistente Datenhaltung: File-based H2 Database für Datensicherheit

Benutzerrollen

Administrator

  • Vollständige Benutzerverwaltung
  • Erstellen und Bearbeiten von Checklisten
  • Durchführen und Verwalten von Inspektionen
  • Zugriff auf alle Berichte

User

  • Erstellen und Bearbeiten von Checklisten
  • Durchführen von Inspektionen
  • Zugriff auf eigene Berichte

Viewer

  • Nur-Lese-Zugriff auf Dashboard
  • Ansicht von Inspektionen und Berichten
  • Keine Bearbeitungsrechte

Workflow

  1. Checkliste erstellen: Admin/User erstellt eine Checkliste mit Prüfpunkten
  2. Inspektion planen: Neue Inspektion mit Anlage, Datum und Checkliste anlegen
  3. Inspektion starten: Status wechselt zu "In Bearbeitung"
  4. Prüfpunkte abarbeiten: Jeder Punkt wird bewertet und dokumentiert
  5. Fotos hochladen: Aktueller Zustand wird fotografisch festgehalten
  6. Inspektion abschließen: Status wechselt zu "Abgeschlossen"
  7. Bericht erstellen: Automatischer Report mit allen Ergebnissen

Installation und Deployment

Voraussetzungen

  • Node.js 16+ und npm
  • Java 17+
  • Gradle

Backend starten

  1. Navigiere zum Backend-Verzeichnis
  2. Führe aus: ./gradlew bootRun
  3. Server läuft auf: http://localhost:8080

Frontend starten

  1. Navigiere zum Frontend-Verzeichnis
  2. Installiere Dependencies: npm install
  3. Starte Development Server: npm start
  4. Anwendung öffnet sich auf: http://localhost:3000
